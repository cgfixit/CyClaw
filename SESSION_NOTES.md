# CyClaw Session Notes

**Purpose:** Track decisions, discoveries, and blockers across Claude Code sessions for continuity and context preservation.

Use this file to record:
- **Session Goals** — what you're trying to accomplish
- **Key Decisions** — architectural choices, tradeoffs, rationale
- **Blockers** — issues that need unblocking before proceeding
- **Discoveries** — important findings about the codebase or behavior
- **Test Coverage Gaps** — areas that need verification
- **Memory Snapshots** — hand-authored facts that should persist across context compaction

---

## Template for Each Session

```markdown
### Session: <YYYY-MM-DD> [Session Title]

**Goal(s):**
- [goal 1]
- [goal 2]

**Key Decisions:**
- [decision 1 — rationale]
- [decision 2 — rationale]

**Blockers:**
- [ ] [blocker 1 — unblocking strategy]

**Discoveries:**
- [finding 1]
- [finding 2]

**Coverage Gaps:**
- [gap 1 — suggested fix]

**Next Action:**
- [action 1]

**Files Modified:**
- `path/to/file.py` — brief change summary
```
